# ispLSI2032 Session — 2026-03-10

## The "Hidden JTAG" Discovery

smooker suspected that the original ispLSI2032's "legacy serial ISP"
protocol is actually JTAG under different names. Research confirmed it.

### Pin Mapping — Same Protocol, Different Names

| Original 2032 | 2032VE | 44-TQFP pin | JTAG Function |
|---------------|--------|-------------|---------------|
| **ispEN** | **BSCAN** | **7** | Enable (active low) |
| **SDI**/IN0 | **TDI**/IN0 | **8** | Test Data In |
| **SDO**/IN1 | **TDO**/IN1 | **18** | Test Data Out |
| **MODE**/NC | **TMS**/NC | **30** | Test Mode Select |
| **SCLK**/Y2 | **TCK**/Y2 | **27** | Test Clock |

### Proof

Lattice patent US5412260A states:

> "SDI, SCLK, MODE and SDO connections required for in-system programming
> and the TDI, TCK, TMS and TDO connections required for boundary-scan
> testing are **multiplexed such that they are provided from the same four
> pins**, with an in-system programming enable pin used to control the
> multiplexing."

### What This Means

The ispLSI 2032-135LT44 found on the Cognex TURBO board (1998) **has JTAG** —
Lattice just didn't market it as IEEE 1149.1 in the original datasheet.
The protocol is the same, the pins are the same.

To use with OpenOCD + UM232H (with 5V to 3.3V level shifter):
1. Hold **ispEN LOW** (activates ISP/JTAG mode)
2. Connect SDI->TDI, SDO->TDO, MODE->TMS, SCLK->TCK
3. Use `irlen 5`

### Donor Board

Cognex TURBO ACR/M/ALRM 2.0, Canada 1998.
Chip marking: `H730B10 / Lattice / ispLSI 2032 / -135LT44`
— 5V, 137 MHz, 44-pin TQFP, commercial grade.
