### 4. The Fuse Map Reverse Engineering Idea

The key insight of the session — **can we reverse-engineer the M4A3 fuse map?**

**smooker's clue**: The BSDL file declares `COLREG[378]` as a programming register.
378 is suspiciously close to 384, and **378 = 384 - 6**.

The 6 subtracted bits correspond to the dedicated clock/input pins that bypass the
programmable AND array. This means COLREG holds exactly one row of AND array fuse
data — 378 programmable connection points per product term.

**The approach** (inspired by Project IceStorm for Lattice iCE40):
1. Write minimal "probe" Verilog designs — one feature per design
2. Run through ispLEVER to get JEDEC (.jed) files
3. Diff JEDEC files between related probes
4. Map fuse bit positions to architectural features
5. Build an open-source fitter — "kato za beli hora"

## Key Numbers for M4A3-64/32

| Parameter | Value |
|-----------|-------|
| PAL blocks | 4 (A, B, C, D) |
| Macrocells per block | 16 |
| Total macrocells | 64 |
| I/O pins | 32 (8 per block) |
| Product terms per macrocell | 5 (borrowable from neighbors) |
| Clock pins | 2 (CLK0/I0, CLK1/I1) |
| JTAG ID | `0x17486157` |
| IR length | 10 bits |
| COLREG width | 378 bits (= 384 - 6 clock/inputs) |
| ROWREG width | 80 bits |
| VCC | 3.0-3.6V |
| Package | 44-pin TQFP (XXVC suffix) |
